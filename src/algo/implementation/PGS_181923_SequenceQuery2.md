## 수열과 구간 쿼리2

### 문제 요약

- 정수 배열 arr와 2차원 정수 배열 queries이 주어짐
- 각 쿼리 [s,e,k]에 대해 arr[s...e] 구간에서 k보다 큰 값들 중 최솟값을 찾는다.
- 해당 값이 없으면 -1을 반환한다.
- 각 쿼리는 서로 독립적으로 처리한다. (이전 쿼리의 결과로 arr가 바뀌지 않음)

### 접근 방식

- 각 쿼리에 대해 구간 s..e를 선형 탐색하며 k보다 큰 값만 고려한다.
- 그중 최솟값 하나만 추적한다.

### 왜 이런 방식으로 풀었나?

- 구간 길이에 비례하는 단순하고 직관적인 방법이며, 문제 제약이 크지 않다면 충분히 빠르다.
- 추가 메모리 사용을 최소화하고(리스트 생략), 조건 비교만으로 답을 얻는다.

### 다른 방식으로 풀었을때 좋은점? (혹은 안좋은점)

- 정렬 / 세그먼트트리 / 오프라인 처리 : 쿼리가 매우 많거나 구간이 길면 고급 자료구조로 k초과 최솟값을 더 빠르게 구할 수 있음
  - 장점 : 대량 쿼리에서 성능 향상 가능
  - 단점 : 구현 복잡도 증가, 이 문제 규모에서는 과한 경우가 많음
