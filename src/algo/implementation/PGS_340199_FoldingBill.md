## 지폐 접기

### 문제 요약

- 지갑 크기 wallet = [w1, w2]와 지폐 크기 bill = [b1, b2]가 주어진다.
- 지폐를 접어 지갑안에 들어가게 만들때 필요한 최소 접기 횟수를 구한다.
- 지폐는 지갑에 그대로 넣거나 90도 회전해서 넣어도 가능하다.
- 한 번 접을때마다 지폐의 긴 변을 반으로 줄인다(정수 나눗셈)

### 접근 방식

- 지갑과 지폐 모두에서 긴변과 작은변을 Math로 정리한다.
- 지폐가 지갑보다 큰 동안 반복한다.
  - 조건 : billBig > walletBig or billSmall > walletSmall
  - 매 반복마다 지폐의 긴변을 나누기 2로 줄인다.
  - 접은 뒤 긴변과 작은변의 관계가 바뀔 수 있으므로, 다시 big/small을 정리한다.
- 접을때마다 횟수 1을 증가시킨다.

### 왜 이런 방식으로 풀었나?

- 지폐는 회전이 가능하므로 결국 큰변끼리 작은변끼리로 들어가는지 판별해야하는 문제이다.
- 문제의 접기 규칙이 긴변을 반으로 줄이기로 고정되어 있으니 매번 긴변만 줄이며 들어갈때까지 반복하며 최소 횟수를 자연스럽게 구할 수 있기때문이다.

### 다른 방식으로 풀었을때 좋은점? (혹은 안좋은점)

- 별도로 완전탐색/DP를 할 필요 없이 규칙을 그대로 시뮬레이션하면 구현이 단순하다.
