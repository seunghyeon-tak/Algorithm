## 공원

### 문제 요약

- 공원은 2차원 배열로 주어진다.
- -1은 빈공간 그 외의 문자열은 사람이 있는 자리이다.
- 여러 크기의 정사각형 돗자리(mats)가 주어진다.
- 돗자리를 빈 공간(-1)에만 깔 수 있다.
- 한 변의 길이가 가장 큰 돗자리를 하나라도 놓을 수 있으면 그 크기를 반환한다.
- 아무 돗자리도 놓을 수 없다면 -1을 반환한다.

### 접근 방식

- mats 배열을 순회하면서 각 돗자리의 크기를 하나씩 검사한다.
- 현재 돗자리 크기(size)에 대해
  - 공원의 모든 칸을 시작점(i, j)으로 잡는다.
  - 해당 위치에서 size x size 영역을 직접 검사한다.
  - 검사 중 하나라도 -1이 아닌 칸이 나오면 실패처리한다.
- 하나라도 성공하는 시작점이 있으면 해당 size는 가능하다.
- 가능한 돗자리 중 가장 큰 값을 반환한다.

### 왜 이런 방식으로 풀었나?

- 문제의 본질은 이 크기의 정사각형이 존재하는가를 확인하는 것이다.
- 최적화가 필수인 문제는 아니라고 생각했으며, 구현 정확도에 중점을 두었다.
- 시작점 선택 -> 내부 영역검사라는 단순한 2단계 구조로 생각하면 2중 + 2중 for문 구조가 된다.
- 부분 영역 조건 검사 문제이므로 브루트포스 방식이 직관적이고 실수 가능성이 적다
