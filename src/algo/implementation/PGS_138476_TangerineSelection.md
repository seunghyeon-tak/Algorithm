## 귤 고르기

### 문제 요약

- 한 상자에 담으려는 귤의 개수 `k`
- 귤의 크기를 담은 배열 `tangerine`
- 귤 `k`개를 고를 때, 서로 다른 크기의 최소 개수를 구하는 문제

### 접근 방식

- 귤의 크기별 개수를 `HashMap`으로 집계한다. 
- 각 크기별 개수만 추출하여 리스트로 만든다.
- 개수를 내림차순 정렬한다.
- 가장 많은 크기부터 차례대로 선택하여 합이 `k` 이상이 될 때까지 누적한다.

### 왜 이런 방식으로 풀었나?

- 종류 수를 최소화하려면
  한 종류에서 최대한 많이 가져가는 것이 유리하다.
- 따라서 가장 개수가 많은 종류부터 선택하는 그리디 전략을 사용했다.
