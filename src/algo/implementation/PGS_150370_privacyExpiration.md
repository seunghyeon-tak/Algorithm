## 개인정보 수집 유효기간

### 문제 요약

- today를 기준으로 개인정보 수집 일자와 약관에 명시된 유효기간을 계산한다.
- 각 개인정보가 유효기간이 지났다면 폐기 대상이 된다.
- 폐기해야 할 개인정보의 번호를 오름차순으로 반환한다.
- 모든 달은 28일로 계산한다는 점이 문제의 핵심 조건이다.

### 접근 방식

- 날짜를 연/월/일 단위로 직접 비교하지 않고,
  **모든 날짜를 ‘총 일 수’로 환산한 뒤 비교**하는 방식을 사용한다.
- 날짜 비교를 단순한 정수 비교로 바꾸면
  월/년 경계 처리나 예외 상황을 따로 고려할 필요가 없다.
- 개인정보 수집일 + (약관 개월 수 × 28일) − 1일을
  해당 개인정보의 만료일로 계산한다.

### 왜 이런 방식으로 풀었나?

- 문제에서 모든 달을 28일로 고정한다고 명시했기 때문에
  날짜를 하나의 숫자 축으로 변환하는 것이 가능하다.
- 연, 월, 일을 각각 처리하는 방식은
  월 초/월 말, 연도 변경 시 예외 처리가 많아진다.
- 날짜를 총 일수로 변환하면
  구현이 단순해지고 논리 오류가 발생할 가능성을 줄일 수 있다.
