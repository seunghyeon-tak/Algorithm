## 문자열 나누기

### 문제 요약

- 문자열 s를 왼쪽에서 오른쪽으로 순차적으로 읽는다.
- 현재 문자열의 첫 문자를 기준 문자 x로 정한다.
- x와 같은 문자 개수와 x가 아닌 문자 개수를 센다.
- 두 개의 개수가 같아지는 순간 하나의 문자열 덩어리로 분리한다.
- 이 과정을 반복했을 때 만들어지는 문자열 덩어리의 개수를 구한다.

### 접근 방식

- 기준 문자를 하나 정하고, 기준 문자와 같은 개수(xCount)와
  기준이 아닌 문자 개수(otherCount)를 각각 센다.
- 문자열을 한 글자씩 읽으면서 두 카운터를 증가시킨다.
- 두 카운터의 값이 같아지는 순간 하나의 문자열 덩어리로 판단하여
  결과 개수를 증가시키고 카운터와 기준 문자를 초기화한다.
- 문자열을 끝까지 처리한 뒤 기준 문자가 남아 있다면,
  그것도 하나의 문자열 덩어리로 카운트한다.

### 왜 이런 방식으로 풀었나?

- 이 문제는 문자열 전체의 빈도를 비교하는 문제가 아니라,
  문자를 하나씩 읽으면서 특정 시점에 분리할 수 있는지를 판단하는 문제라고 생각했다.
- 따라서 기준 문자를 중심으로 같은 문자와 다른 문자의 개수를
  즉시 비교하는 방식이 문제의 조건을 가장 직관적으로 만족한다고 판단했다.

### 다른 방식으로 풀었을때 좋은점? (혹은 안좋은점)

- Map이나 문자열 전체 빈도를 사용하는 방식도 시도할 수 있지만,
  분리 시점이 중요한 이 문제에서는 오히려 로직이 복잡해질 수 있다.
- 현재 방식은 두 개의 카운터만 사용하므로 구현이 단순하고,
  문제의 흐름을 그대로 코드로 옮길 수 있다는 장점이 있다.
