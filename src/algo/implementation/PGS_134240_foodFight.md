## 푸드 파이트 대회

### 문제 요약

- 음식 종류별 개수가 담긴 배열 `food`가 주어진다.
- 두 선수가 먹는 순서를 하나의 문자열로 구성해야 한다.
- 각 선수는 동일한 양의 음식을 먹어야 하므로,
  각 음식 개수의 **절반만** 사용해 양쪽에 대칭 형태로 배치한다.
- 가운데에는 항상 물(0)을 배치한다.

### 접근 방식

- 음식 번호 1부터 시작하여, 각 음식의 개수 food[i]를 2로 나눈 값(food[i] / 2)만큼 왼쪽 선수의 문자열에 추가한다.
- 이렇게 만들어진 문자열에 0을 추가하고 오른쪽 선수의 문자열은 reverse()하여 문자열을 뒤집퍼서 붙인다.

### 왜 이런 방식으로 풀었나?

- 두 선수는 완전히 동일한 양과 순서로 음식을 배치해야 하기 때문에
- 음식 갯수의 절반만 사용하면 대칭 구조를 쉽게 만들 수 있다.
- 가운데는 물(0)로 고정되므로, 대칭 구조를 이어주는 역할을 한다.
- StringBuilder를 사용하면 대칭 구조 문자열을 효율적으로 만들 수 있다.

### 다른 방식으로 풀었을때 좋은점? (혹은 안좋은점)

- String.repact() 사용 방식
```java
StringBuilder left = new StringBuilder();

for (int i = 1; i < food.length; i++) {
    int count = food[i] / 2;
    left.append(String.valueOf(i).repeat(count));
    }

String answer = left.toString() + "0" + left.reverse().toString();
```
- 장점
  - String.repact(n)을 사용하면 반복문을 직접 돌리지 않아도 되어 코드가 매우 간결해진다.
- 단점
  - Java11 이상에서만 사용가능
  - 반복문 방식에 비해 내부적으로 더 많은 문자열에 객체를 생성할 수 있어 성능은 조금 떨어질 수 있다.
