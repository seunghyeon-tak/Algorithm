## 대충 만든 자판

### 문제 요약

- 휴대폰 자판은 하나의 키를 여러 번 눌러 문자를 입력하는 방식이다.
- `keymap`은 각 자판에 할당된 문자열을 의미하며,  
  문자열의 인덱스 + 1이 해당 문자를 입력하는 데 필요한 횟수이다.
- `targets`에 주어진 문자열을 만들기 위해 필요한 **최소 입력 횟수**를 구한다.
- 만약 만들 수 없는 문자열이라면 `-1`을 출력한다.


### 접근 방식

1. `keymap`을 먼저 순회하며  
   각 문자에 대해 **입력 가능한 최소 횟수**를 `HashMap`에 저장한다.
2. 동일한 문자가 여러 keymap 문자열에 존재할 수 있으므로  
   이미 존재하는 경우에는 기존 값과 비교하여 더 작은 값을 유지한다.
3. 이후 `targets`의 각 문자열을 순회하면서  
   문자 단위로 map에서 값을 조회하여 누적 합을 계산한다.
4. 만약 map에 존재하지 않는 문자가 등장하면  
   해당 문자열의 결과는 `-1`로 처리한다.

### 왜 이런 방식으로 풀었나?

- `keymap`을 매번 탐색하는 방식은 중첩 반복문이 많아져 비효율적이기 때문에,
- 미리 문자별 최소 입력 횟수를 정리해두는 방식이
  시간 복잡도와 구현 난이도 측면에서 가장 직관적이라고 판단했다.
- `HashMap`을 사용하면 문자 조회를 O(1)에 처리할 수 있어
  전체 로직을 단순하게 유지할 수 있다.

### 다른 방식으로 풀었을때 좋은점? (혹은 안좋은점)

- targets의 각 문자마다 keymap 전체를 탐색하는 방식도 가능하지만,
  구현이 복잡해지고 불필요한 반복이 많아진다.
- 현재 방식은 전처리 후 빠른 조회가 가능하여
  가독성과 성능 모두에서 장점이 있다.
