## 콜라 문제

### 문제 요약

- 정수 n병의 콜라병을 가지고 있다.
- 마트에서는 빈병 a개 -> 새 콜라 b개로 교환해준다.
- 교환을 더 이상 할 수 없을때까지 반복할 때 총 몇병의 콜라를 마시게 되는지 계산하는 문제

### 접근 방식

- 현재 가진 빈 병 수 n이 a이상일 때만 교환 가능 while(n >= a) 조건으로 반복한다.
- 한 번의 반복턴에서 해야할 계산
  - 교환에 사용할 수 있는 빈 병수
    - used = (n / a) * a
  - 이번 턴에서 새로 받을 수 있는 콜라 수
    - received = (n / a) * b
- 받은 콜라는 즉시 마신것으로 간주 answer += received로 누적됨
- 다음 턴에서 사용할 빈병의 개수 업데이트
  - used는 사라지고, received는 콜라를 마신 후 다시 빈 병으로 돌아오므로
  - n = n - used + received
- n이 a보다 작아지면 더 이상 교환이 불가능하므로 종료

### 왜 이런 방식으로 풀었나?

- 해당 문제는 교환 과정으로 직접 시뮬레이션하는 문제이기에
- 한번의 교환으로 빈병의 개수가 계속 변하기 때문에 고정된 반복 횟수가 존재하지 않는다.
- 나눗셈을 통해 한 턴에 가능한 교환 횟수를 빠르게 계산할 수 있고
- 이 과정을 while문으로 반복하면 상태 변화를 자동으로 처리할 수 있다.
