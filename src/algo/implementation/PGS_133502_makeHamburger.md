## 햄버거 만들기

### 문제 요약

- 햄버거는 **빵(1) → 야채(2) → 고기(3) → 빵(1)** 순서로 만들어진다.
- `ingredient` 배열에 주어진 재료들을 **순서대로 사용**하여
  만들 수 있는 햄버거의 **총 개수**를 구하는 문제이다.
- 재료는 앞에서부터 하나씩 사용하며, 중간에 건너뛰거나 순서를 바꿀 수 없다.


### 접근 방식

- 햄버거는 항상 연속된 4개의 재료로만 완성되므로,
  재료를 하나씩 쌓아가며 최근에 쌓인 재료 4개만 확인한다.
- 이를 위해 결과 상태를 저장할 배열(List)을 하나 유지하고,
  재료를 순서대로 뒤에 추가한다.
- 배열의 크기가 4 이상일 때,
  뒤에서부터 4개의 재료가 1, 2, 3, 1 패턴인지 확인한다.
- 패턴이 일치하면 해당 4개의 재료를 제거하고
  햄버거 개수를 1 증가시킨다.

### 왜 이런 방식으로 풀었나?

- 햄버거는 항상 마지막에 추가된 재료를 포함해 만들어지므로,
  전체를 탐색할 필요 없이 마지막 상태만 확인하면 된다.
- 결과 배열을 스택처럼 사용하면
  재료의 순서를 자연스럽게 유지할 수 있고,
  조건이 만족될 때 뒤에서만 제거하면 되기 때문에
  구현이 단순하고 안정적이다.
- 불필요한 중첩 반복문을 피할 수 있어
  시간 복잡도 측면에서도 효율적이다.

### 다른 방식으로 풀었을때 좋은점? (혹은 안좋은점)

- Stack 또는 ArrayDeque를 사용해
  스택 자료구조로 구현할 수도 있다.
- ArrayDeque 사용 방법
```java
class Solution {
    public int solution(int[] ingredient) {
        int answer = 0;
        Deque<Integer> deque = new ArrayDeque<>();

        for (int i : ingredient) {
            deque.addLast(i);

            if (deque.size() >= 4) {
                Integer[] arr = deque.toArray(new Integer[0]);
                int n = arr.length;

                if (arr[n - 4] == 1 &&
                    arr[n - 3] == 2 &&
                    arr[n - 2] == 3 &&
                    arr[n - 1] == 1) {

                    // 뒤에서 4개 제거
                    for (int k = 0; k < 4; k++) {
                        deque.removeLast();
                    }
                    answer++;
                }
            }
        }
        return answer;
    }
}
```
