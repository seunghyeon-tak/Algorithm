## 신고 결과 받기

### 문제 요약

- 유저가 다른 유저를 신고할 수 있다.
- 동일한 유저를 여러 번 신고하더라도 1회만 인정된다.
- k번 이상 신고된 유저는 정지된다.
- 정지된 유저를 신고한 사람에게 메일이 발송된다.
- 각 유저가 받는 메일의 개수를 id_list 순서대로 출력한다.

### 접근 방식

- 중복 제거
  - Map<신고자, Set<피신고자>> 구조 사용
  - Set을 이용해 동일 신고 제거
- 신고당한 횟수 계산
  - Map<피신고자, 신고당한횟수> 생성
- 정지 재상 선별
  - Set<정지된 사용자> 생성
- 메일 수 계산
  - id_list 순서대로
  - 자신이 신고한 사람 중 정지된 사람 수 카운트

### 왜 이런 방식으로 풀었나?

- 중복 제거가 핵심이므로 Set 사용이 필수
- 신고자 / 피신고자 관계를 명확히 분리하기 위해 Map 사용
- 순서 보장을 위해 마지막은 반드시 id_list 기준으로 순회
